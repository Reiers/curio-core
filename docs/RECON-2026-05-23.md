## Surprising findings

1. **PDP imports `market/mk20` and `market/ipni/ipniculib`.** I expected the market dep was strippable; it isn't. mk20 is the deal-protocol type definitions used by PDP for piece lookups. Won't fight this; vendor the small subset.

2. **PDP imports `lib/ffi`** but uses only `SealCalls.PieceReader`. The CGo dependency is hard to remove cleanly because `SealCalls` is the type used; need to either:
   a. Define a `PieceReadProvider` interface in curio-core and inject our pure-Go impl. ← preferred
   b. Vendor `lib/ffi` minus the CGo functions. ← uglier
   c. Build with `-tags pure_go` and rely on upstream Curio building stub fns. ← needs upstream changes

   Picking (a). One interface, one provider, no CGo.

3. **`integ/task` harmonytask is significantly different from main**. The doc says "single-threaded scheduler on the decision path" + "peer-to-peer coordination" + "DB for authoritative claims." For curio-core single-node we run the scheduler + DB only. Peering layer is a clean compile-out.
